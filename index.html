Cards work without JS and allow for different interaction triggers with classes .hover .tap .open.

Additionally for floating width cards you can use class .flexible when width is configured by the grid or other elements.

<div class="card" tabindex="0">
	<div class="card-image">
		<img src="">
	</div>
	<div class="card-title">
		Title
	</div>
	<div class="card-content">
		Content
	</div>
</div>

Alerts can be used for various information displays anywhere on a page, an alert dispetcher module for JS is currently in the works.

Colours are configured through classes .info .success .warning .danger or can be left by default with backdrop shadow added through .shaded class.

<div class="alert">
	<span class="dismiss-btn" dismiss>&times;</span>
</div>

Forms

Standard inputs


All form input types are supported and made uniform using one class .form-input.

Any input, select and textarea element with this class will be made fit to width of its parent as well as given uniform flat look.

Fancy inputs


This is an extension to standard forms and any form element can be made into a "fancy" one using .form-fancy class.

However conversion is done via JS on page load thus unsupported clients will fall back to regular style.

Buttons

You can use both a and button elements for buttons, note however that buttons must have type property of a button.

All same colouring calsses are supported as with alerts, namely .info .success .warning and .danger. 

<a class="button">Button</a>

Inputs with buttons

You can add buttons on both sides of inputs for gapless context buttons or info-labels (like what type of input this is)

Keeping up with supporting all form elements are buttons on both sides adjacent to the inputs.

A class .left should be added if your button should be on the left and .right for the right side.

Note however that only one button per side is currently supported. 

<div class="form-input-combo">
	<a class="button left">ABC</a>
	<input class="form-input">
	<a class="button right">ABC</a>
</div>

Tables

Included with hailstorm are flexible grids with a variety of styles for better readability.

You can see in an example a table with all styles applied.

Namely .striped for row-to-row highlighting, .flexible to fill width of container and .hover for highlighting individual rows when hovering.

<table class="table striped flexible hover">...</table>
# Title
1 Lorem ipsum
2 Dolor
3 Sit amet
# Title

Navigation

Very simple pre-made navigation is available, you can additionally use the grid to space out the navigation to your liking.

Use class .inverted for white-on-black look, .sticky for navbar to stick to page top after scrolling past it or .floating to affix it to screen top. 

<nav class="nav">
	<ul>
		<li class="active"><a href="#">Home</a></li>
	</ul>
</nav>
